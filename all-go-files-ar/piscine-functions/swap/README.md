# شرح `swap.go` — تبديل قيمتين باستخدام المؤشرات

> الدالة الأساسية: `Swap`

## ما وظيفة الكود؟

تبدل القيمتين الأصليتين اللتين يشير إليهما a وb باستخدام متغير مؤقت.

## الكود الأصلي

```go
package piscine

func Swap(a *int, b *int) {
	temp := *a
	*a = *b
	*b = temp
}
```

## المدخلات والمخرجات

```go
func Swap(a *int, b *int)
```

الدالة تستقبل القيم حسب التوقيع أعلاه، ثم إما تعيد نتيجة بواسطة `return` أو تعدّل قيمة عبر pointer أو تطبع باستخدام `z01.PrintRune`.

## تدفق التنفيذ خطوة بخطوة

1. حفظ قيمة *a في temp.
2. كتابة قيمة *b داخل *a.
3. كتابة temp داخل *b.

## تتبع بأمثلة

- `x=3,y=7; Swap(&x,&y)` ⟶ `x=7,y=3`

## المفاهيم المهمة

- **pointers**
- **temporary variable**

## شرح كل سطر

| السطر | التعليمة | الشرح |
|---:|---|---|
| 1 | `package piscine` | يحدد الحزمة التي ينتمي إليها الملف؛ `piscine` تعني أن الدالة جزء من مكتبة التمارين، و`main` تعني برنامجًا قابلًا للتشغيل. |
| 3 | `func Swap(a *int, b *int) {` | تعريف دالة `Swap` مع المعاملات ونوع الإرجاع الظاهرين في التوقيع. |
| 4 | `temp := *a` | إنشاء المتغير `temp` لأول مرة واستنتاج نوعه تلقائيًا من القيمة الموجودة يمين `:=`. |
| 5 | `*a = *b` | يكتب قيمة جديدة داخل المتغير الأصلي عبر فك المؤشر باستخدام `*`. |
| 6 | `*b = temp` | يكتب قيمة جديدة داخل المتغير الأصلي عبر فك المؤشر باستخدام `*`. |
| 7 | `}` | نهاية الكتلة الحالية. |

## الحالات الخاصة والأخطاء الشائعة

- المؤشرات nil تسبب panic.
- إذا أشار المؤشران إلى المتغير نفسه فلن تتغير القيمة.

## التعقيد

O(1).

## طريقة تجربة الدالة

لأن الملف يستخدم `package piscine` فهو ليس برنامجًا مستقلًا. أنشئ ملف `main.go` مؤقتًا في مجلد آخر يستورد الحزمة، أو استخدم ملف اختبار Go. مثال اختبار مبسط:

```go
package piscine

import "testing"

func TestSwap(t *testing.T) {
	// ضع هنا حالات الاختبار المناسبة كما في الأمثلة أعلاه.
}
```

> لا تضف ملف `main.go` داخل مجلد التسليم إذا كانت منصة التمرين تطلب ملف الدالة فقط.
