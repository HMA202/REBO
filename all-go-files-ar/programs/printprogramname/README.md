# شرح تمرين printprogramname

## فكرة التمرين

المطلوب كتابة برنامج يطبع **اسم الملف التنفيذي الذي تم تشغيله**، وليس اسم المجلد ولا المسار الكامل.

أمثلة:

```bash
go build main.go
./main
```

الناتج:

```text
main
```

وعند اختيار اسم مختلف للملف التنفيذي:

```bash
go build -o Nessy
./Nessy
```

الناتج:

```text
Nessy
```

---

## الملف المطلوب للتسليم

```text
printprogramname/main.go
```

## الدوال المسموحة

- `os.Args`
- `z01.PrintRune`
- الدوال والعمليات المضمنة المسموحة بواسطة `--allow-builtin`

---

## الكود

```go
package main

import (
    "os"

    "github.com/01-edu/z01"
)

func main() {
    programPath := os.Args[0]
    start := 0

    for i, r := range programPath {
        if r == '/' {
            start = i + 1
        }
    }

    for _, r := range programPath[start:] {
        z01.PrintRune(r)
    }

    z01.PrintRune('\n')
}
```

## الشرح التفصيلي

### 1. لماذا نستخدم `package main`؟

```go
package main
```

لأن التمرين يطلب **برنامجًا قابلًا للتشغيل**. أي برنامج Go تنفيذي يجب أن يكون داخل الحزمة `main` ويحتوي على دالة اسمها `main`.

### 2. استيراد `os`

```go
import "os"
```

نستخدم الحزمة `os` للوصول إلى:

```go
os.Args
```

`os.Args` هي قائمة من النصوص من النوع:

```go
[]string
```

العنصر الأول فيها، أي:

```go
os.Args[0]
```

يحتوي على الاسم أو المسار المستخدم لتشغيل البرنامج.

مثال:

```bash
./main
```

قد تكون قيمة `os.Args[0]`:

```text
./main
```

وعند التشغيل من مسار أطول قد تكون:

```text
/home/student/printprogramname/main
```

لذلك لا يكفي أن نطبع `os.Args[0]` مباشرة، لأن المطلوب هو `main` فقط.

### 3. حفظ مسار البرنامج

```go
programPath := os.Args[0]
```

أنشأنا متغيرًا اسمه `programPath` ووضعنا فيه المسار الذي استُخدم لتشغيل البرنامج.

### 4. تحديد بداية اسم البرنامج

```go
start := 0
```

المتغير `start` سيحفظ رقم البايت الذي يبدأ عنده اسم الملف التنفيذي.

نبدأ بالقيمة `0` تحسبًا لأن `os.Args[0]` قد يكون اسمًا فقط، مثل:

```text
Nessy
```

في هذه الحالة لا يوجد `/`، ويجب أن تبدأ الطباعة من أول النص.

### 5. البحث عن آخر `/`

```go
for i, r := range programPath {
    if r == '/' {
        start = i + 1
    }
}
```

تمر حلقة `range` على كل حرف في `programPath`.

- `i` هو موقع بداية الحرف داخل النص بالبايت.
- `r` هو الحرف الحالي من نوع `rune`.

كلما وجدنا `/` نحدّث `start` ليصبح الموقع الذي بعده:

```go
start = i + 1
```

لم نستخدم `break`؛ لأننا نريد **آخر** `/` وليس أول `/`.

مثال:

```text
/home/student/main
```

توجد عدة شرطات `/`. في كل مرة تتغير قيمة `start`، وفي النهاية تصبح بعد آخر `/` مباشرة، أي عند الحرف `m` من `main`.

### 6. لماذا استُخدم `range` بدل الفهرسة العادية؟

يمكن قراءة نص إنجليزي بهذه الطريقة:

```go
programPath[i]
```

لكنها تعيد `byte` واحدًا، وليس بالضرورة حرف Unicode كاملًا. قد يختبر النظام اسمًا يحتوي على إيموجي أو أحرف غير إنجليزية، مثل:

```text
🤦🏻‍♀️
```

لذلك نطبع بواسطة:

```go
for _, r := range programPath[start:] {
    z01.PrintRune(r)
}
```

وبذلك يقرأ Go كل حرف Unicode على هيئة `rune` صحيح بدل طباعة البايتات بشكل مشوّه.

### 7. طباعة الاسم

```go
for _, r := range programPath[start:] {
    z01.PrintRune(r)
}
```

هذا الجزء يقتطع النص ابتداءً من `start` وحتى النهاية، ثم يطبع كل `rune` باستخدام `z01.PrintRune`.

### 8. السطر الجديد

```go
z01.PrintRune('\n')
```

يطبع نهاية سطر. وهو ضروري حتى يظهر الاختبار باستخدام `cat -e` هكذا:

```text
printprogramname$
```

علامة `$` هنا ليست من البرنامج، بل يعرضها `cat -e` للدلالة على وجود newline.

## تتبع مثال خطوة بخطوة

لو كانت قيمة:

```text
programPath = "./Nessy"
```

الأحرف المهمة:

| الموقع | الحرف | الإجراء |
|---:|:---:|---|
| 0 | `.` | لا شيء |
| 1 | `/` | تصبح `start = 2` |
| 2 | `N` | لا شيء |
| 3 | `e` | لا شيء |
| 4 | `s` | لا شيء |
| 5 | `s` | لا شيء |
| 6 | `y` | لا شيء |

بعد انتهاء الحلقة:

```go
programPath[start:]
```

يساوي:

```text
Nessy
```

## أخطاء شائعة

1. طباعة `os.Args[0]` مباشرة، فيظهر `./main` بدل `main`.
2. استخدام `programPath[i]` ثم تحويل كل بايت إلى `rune`، وهذا يشوّه أسماء Unicode.
3. نسيان `\n` في نهاية الناتج.
4. عدم وجود newline بعد آخر `}` في الملف، فيفشل `gofumpt`.
5. استخدام `package piscine` بدل `package main`.
