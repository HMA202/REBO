# شرح تمرين sortparams

## فكرة التمرين

المطلوب ترتيب arguments حسب ترتيب ASCII ثم طباعة كل argument في سطر منفصل.

مثال:

```bash
go run . 1 a 2 A 3 b 4 C
```

الناتج:

```text
1
2
3
4
A
C
a
b
```

الأرقام تأتي قبل الحروف الكبيرة، والحروف الكبيرة تأتي قبل الحروف الصغيرة في ASCII.

---

## الكود

```go
package main

import (
    "os"

    "github.com/01-edu/z01"
)

func main() {
    args := os.Args[1:]

    for i := 0; i < len(args)-1; i++ {
        for j := 0; j < len(args)-1-i; j++ {
            if args[j] > args[j+1] {
                args[j], args[j+1] = args[j+1], args[j]
            }
        }
    }

    for _, argument := range args {
        for _, r := range argument {
            z01.PrintRune(r)
        }

        z01.PrintRune('\n')
    }
}
```

---

## أخذ arguments فقط

```go
args := os.Args[1:]
```

نستبعد `os.Args[0]` لأنه اسم البرنامج.

`args` ليست نسخة جديدة منفصلة تمامًا؛ بل Slice تشير إلى جزء من المصفوفة الأصلية. لذلك عندما نبدل عناصر `args` فإن ترتيب الجزء المقابل في `os.Args` يتغير أيضًا، وهذا لا يسبب مشكلة هنا.

## المقارنة بين strings في Go

الشرط:

```go
if args[j] > args[j+1]
```

يقارن النصين ترتيبًا lexicographical، أي حرفًا بحرف وفق قيم البايتات/Unicode. بالنسبة إلى مدخلات ASCII يحقق المطلوب.

قيم شهيرة:

```text
'0' = 48
'1' = 49
'A' = 65
'C' = 67
'a' = 97
'b' = 98
```

لذلك:

```text
"1" < "A" < "a"
```

وبالنسبة للكلمات، تتم المقارنة من أول حرف مختلف:

```text
"apple" < "banana"
"car" < "cat"
```

وفي المثال الثاني يتساوى `c` ثم `a`، وبعدها تتم مقارنة `r` مع `t`.

## خوارزمية Bubble Sort

الكود يستخدم Bubble Sort، وهي خوارزمية تقارن كل عنصر بالعنصر المجاور له وتبدلهما إذا كانا بترتيب خاطئ.

### الحلقة الخارجية

```go
for i := 0; i < len(args)-1; i++ {
```

تحدد عدد جولات الترتيب. لقائمة طولها `n` نحتاج بحد أقصى `n-1` جولات.

### الحلقة الداخلية

```go
for j := 0; j < len(args)-1-i; j++ {
```

تقارن العناصر المتجاورة:

```go
args[j]
args[j+1]
```

نطرح `i` لأن كل جولة تضع أكبر عنصر متبقٍ في آخر الجزء غير المرتب، فلا نحتاج إعادة فحصه.

### التبديل

```go
args[j], args[j+1] = args[j+1], args[j]
```

Go تسمح بتبديل متغيرين في سطر واحد دون متغير مؤقت.

قبل التبديل:

```text
a A
```

بعد التبديل:

```text
A a
```

---

## تتبع مصغر

القائمة:

```text
b A 1
```

### الجولة الأولى

1. مقارنة `b` و`A`: يتم التبديل → `A b 1`
2. مقارنة `b` و`1`: يتم التبديل → `A 1 b`

### الجولة الثانية

1. مقارنة `A` و`1`: يتم التبديل → `1 A b`

أصبحت مرتبة.

---

## طباعة النتائج

```go
for _, argument := range args {
    for _, r := range argument {
        z01.PrintRune(r)
    }

    z01.PrintRune('\n')
}
```

الحلقة الخارجية تمر على arguments بعد الترتيب. الحلقة الداخلية تطبع أحرف كل argument. ثم يطبع newline.

---

## التعقيد الزمني

Bubble Sort تستخدم حلقتين متداخلتين، لذلك التعقيد في أسوأ حالة:

```text
O(n²)
```

حيث `n` عدد arguments. هذا مقبول في هذا التمرين لأن عدد arguments عادة صغير، ولأن استخدام مكتبات ترتيب جاهزة قد لا يكون مسموحًا.

## أخطاء شائعة

1. ترتيب الأحرف داخل كل argument بدل ترتيب arguments.
2. إدخال `os.Args[0]` في الترتيب.
3. استخدام `j+1` دون ضبط نهاية الحلقة، مما يسبب index out of range.
4. افتراض أن الحروف الصغيرة تأتي قبل الكبيرة.
5. طباعة arguments في سطر واحد بدل سطر لكل argument.
